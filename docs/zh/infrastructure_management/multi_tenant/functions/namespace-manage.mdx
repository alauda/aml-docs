# 命名空间管理

AML 支持的租户是 K8s 命名空间，这些命名空间在用作 AML 的租户之前需要在 **命名空间管理** 中进行管理。

创建命名空间并将用户添加到该命名空间的步骤如下：

1. 以管理员身份登录到 ACP 平台

2. 创建命名空间
   1. 进入 ACP **项目管理**
   2. 创建新项目或选择现有项目，然后点击进入当前项目的详细页面。
   3. 点击 **命名空间** > **命名空间**
   4. 点击 **创建命名空间**
   5. 在 **创建命名空间** 页面中，在 **Pod 安全策略** 部分，选择 **privileged** 作为所有三个 `enforce`、`audit` 和 `warn` 安全模式的安全标准。

      :::warning

      安全标准必须为空或 **privileged**，否则 AI 推理工作负载可能拒绝运行。

      :::

3. 将命名空间权限分配给用户
   1. 进入 ACP **平台管理**
   2. 从导航中选择 **用户角色管理** > **用户管理**
   3. 创建新用户或选择要添加至该命名空间的用户
   4. 点击 **配置角色** 标签，然后点击 **添加角色** 按钮
   5. 为该用户配置与 AML 相关的角色，并选择之前创建的命名空间
      - 编辑者：用户可以在命名空间中创建和删除工作负载，并使用资源配额，例如 CPU 和 GPU，以完成工作
      - 拥有者：用户有权限管理该 AML 命名空间
      - 查看者：用户只能查看命名空间的资源，不能创建或删除工作负载

4. 返回 AML，点击 **平台管理** > **命名空间管理** 中的 **管理命名空间**，选择之前创建的命名空间以完成管理
