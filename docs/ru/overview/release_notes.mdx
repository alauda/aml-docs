# Примечания к релизу

## AI 1.3.0

### **Новые и оптимизированные функции**

#### Ограничение доступа к общим моделям

Репозиторий моделей в настоящее время поддерживает два типа моделей: **Общие модели** и **Частные модели**. В первоначальном проекте пользователи могли выполнять некоторые действия управления над общими моделями, что создавало потенциальные риски доступа.\
В этом релизе функциональность и разрешения для Частных моделей остались без изменений, поддерживая полные операции управления. **Разрешения для Общих моделей ограничены и оптимизированы** следующим образом:

- Ограничение доступа: Все пользователи могут использовать только общие модели. Возможность создания, редактирования или удаления общих моделей больше не поддерживается.
- Корректировка процесса создания: Параметр видимости удален из процесса "Создать модель". Все вновь созданные модели по умолчанию становятся **Частными моделями**.
- Удаление функций:
  - В следующем функционале для Общих моделей:
    - Кнопка редактирования тегов
    - Кнопка редактирования описания
    - Кнопка создания тега
    - Кнопка удаления
    - Вкладка управления файлами
    - Вкладка управления версиями

#### Новая публикация шаблонов для сервисов вывода

Ранее создание сервисов вывода требовало ручной настройки множества взаимозависимых параметров. Эта сложность часто приводила к ошибкам, снижая уровень успешности и влияя на пользовательский опыт.\
В этом релизе представлена функция **Публикации шаблонов**, позволяющая пользователям инкапсулировать проверенные конфигурации в качестве шаблонов и быстро публиковать сервисы вывода на их основе.\
Преимущества включают:

- Пользователи могут создавать собственные шаблоны, повторно используя проверенные лучшие практики.
- Автозаполнение параметров конфигурации уменьшает повторный ввод и ошибки зависимости.
- Снижает барьер для публикации сервисов вывода больших моделей, улучшая уровень успешности и эффективность.

#### Поддержка Multi-GPU на одном узле для времени вывода

Ранее сервисы вывода, развернутые на одном узле, поддерживали только режим **одного GPU** из-за ограничений распределения ресурсов. Это ограничивало сценарии вывода больших моделей и неэффективно использовало ресурсы GPU.\
С помощью этого обновления теперь поддерживается **распределение multi-GPU** внутри одного узла. Один сервис вывода может автоматически распределять несколько GPU на одной машине, позволяя выполнять вывод больших моделей, улучшая использование ресурсов и повышая возможности сервиса.

#### "Мониторинг бизнеса" для сервисов вывода

Ранее сервисы вывода отображали только базовую информацию. Для улучшения наблюдаемости и обеспечения быстрого выявления проблем пользователями, мониторинга состояния сервиса в реальном времени и проактивной оптимизации или настройки ресурсов, вводится следующая новая функция:\
**Панель мониторинга**

- Добавлена в качестве новой вкладки в сервисах вывода, охватывающей три измерения:
  - **Мониторинг ресурсов**: использование CPU (ядра), загрузка CPU (%), использование памяти (GiB), загрузка памяти (%)
  - **Мониторинг вычислений**: использование GPU (ядра), загрузка GPU (%), использование памяти GPU (GiB), загрузка памяти GPU (%)
  - **Другие метрики**: Время ответа, Трафик (входящий/исходящий объем данных), QPS (Запросы в секунду), Всего вызовов, Пропускная способность токенов (/с)

#### Расширение времени вывода

Чтобы улучшить поддержку времени вывода AML, в этой версии добавлены следующие новые среды выполнения:

- vllm-cuda-11.8
- vllm-cpu

#### Выделенный "Вид управления платформой"

Ранее функции управления платформой (включая управление пространствами имен и управление учетными данными) были смешаны с бизнес-функциями в одном представлении, что вызывало путаницу из-за смешивания уровней разрешений.\
В этом релизе:

- **Функции управления платформой отделены в независимое представление**, видимое и управляемое **только администраторами**.
- Администраторы могут свободно переключаться между **"Видом управления"** и **"Бизнес-видом"** через верхнее навигационное меню.
- Обычные пользователи могут получить доступ только к **Бизнес-ведению**, без доступа к функциям управления платформой.

#### Автоматическая конфигурация токена GitLab для внедрения пространств имен

Ранее при внедрении пространства имен пользователи должны были вручную настраивать токен GitLab для авторизации доступа к репозиторию.\
Этот релиз оптимизирует процесс авторизации GitLab, реализуя **автоматическую конфигурацию токена GitLab**:

- Для каждого вновь внедренного пространства имен платформа автоматически настраивает токен GitLab.
- Пользователям не требуется никаких ручных операций или управления авторизацией GitLab.
- Обеспечен непрерывный доступ к GitLab для всех управляемых пространств имен.

### **Устаревшие функции**

#### Понижение α функций до стадии S2

В ходе итераций платформы AML некоторые модули были выпущены как **α функции** для экспериментальной валидации дизайнов и потребностей пользователей.\
Однако, из-за быстрых изменений в сценариях разработки больших моделей и развивающихся требований пользователей, некоторые α функции имеют недостатки дизайна или ограниченную применимость. Эти функции будут **переоценены** и **понижены до стадии S2** для дальнейшего планирования.\
Пониженные функции:

- **Набор данных**: Репозиторий наборов данных, Разметка данных
- **Оптимизация модели**: Шаблоны задач, Тонкая настройка модели, Предобучение
- **Агенты**: Репозиторий приложений, Dify
- **Расширенные функции**: Блокнот, Объемы хранения, MLFlow, Tensorboard, Рабочий процесс, Задачи рабочего процесса, Запланированные задачи, AutoML
- **Модель**: Создание образа API вывода
