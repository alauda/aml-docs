# DeepSeek-R1-Distill-Llama-8B (dense)

Dense 8B model (Llama 3.1 8B base, R1-reasoning distilled, BF16). Validated on a single
Ascend 910B4 card at TP=1 with vLLM-Ascend.

## Model identity

| Field | Value |
|---|---|
| Publisher | DeepSeek (`deepseek-ai`) |
| Architecture | `LlamaForCausalLM` — 32 layers / 4096 hidden / 14336 intermediate / SwiGLU / GQA 32-8 |
| Parameters | 8B (Llama 3.1 8B base, R1-reasoning distilled) |
| Native dtype | BF16 |
| Hugging Face | https://huggingface.co/deepseek-ai/DeepSeek-R1-Distill-Llama-8B |

> Why the distilled build instead of base Llama 3.1 8B: `R1-Distill` is published directly by
> `deepseek-ai` (no gate) and is architecturally identical to Llama 3.1 8B, so it is a clean
> "mature path" baseline.

## Validated hardware × stack

| Platform | Engine | Version | Status |
|---|---|---|---|
| Ascend 910B4 × 1 (TP=1, whole card) | vLLM-Ascend | nightly-main-openeuler (vllm v0.20.2 + CANN 9.0.0) | ✅ chatbot 524/256, clean single-stream data |

## Deploy

| Config | File |
|---|---|
| vLLM-Ascend, TP=1 | [`deepseek-r1-distill-llama-8b-vllm-ascend-tp1.yaml`](https://github.com/alauda/aml-docs/tree/master/docs/en/inference_guide/assets/deepseek-r1-distill-llama-8b/deepseek-r1-distill-llama-8b-vllm-ascend-tp1.yaml) |

```bash
base=https://raw.githubusercontent.com/alauda/aml-docs/master/docs/en/inference_guide/assets/deepseek-r1-distill-llama-8b
# edit namespace / image tag / storageUri first, then:
kubectl apply -f $base/deepseek-r1-distill-llama-8b-vllm-ascend-tp1.yaml
```

Single-card memory budget (validated): 16 GB BF16 weights on a 32 GB card → ~11 GB KV
headroom at `--gpu-memory-utilization 0.85` → ~20 concurrent sequences at
`--max-model-len 4096`, enough for chat.

## Benchmark results

Validated single-stream on the chatbot workload (524/256, TP=1, 910B4). The per-rate sweep
figures for this model were captured as part of a cross-run comparison rather than a
standalone open-loop report, so no per-replica RPS table is published here.
