## Tuning notes

1. **W4A8 path works** â€” Ascend ModelSlim W4A8 + vLLM-Ascend inference serves the full 755B-class
   model on one 8-card node (~391 GB weights at W4A8).
2. **The concurrency cap is configuration, not hardware.** Median concurrency stays at 2.0 across
   the whole sweep because `--max-num-seqs=2`; a single stream reads weights at only ~1.6% of the
   6.4 TB/s aggregate HBM bandwidth, and the 6.36 GiB/card KV cache holds ~61k tokens. Raising
   `--max-num-seqs` is the first throughput lever (revalidate before committing a higher value).
3. **`--enforce-eager` over compiled kernels.** It is the actual fix for the â‰Ą512-token prefill
   crash; it follows upstream precedent (the same flag is used on prefill nodes in PD
   disaggregation). The decode path loses cudagraph, which is why ITL is slightly elevated.
