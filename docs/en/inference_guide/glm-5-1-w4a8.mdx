# GLM-5.1-W4A8 (MoE, quantized)

W4A8-quantized MoE (GLM-5.1, ~755B total / ~42B active; 4-bit weights + 8-bit activations via
Ascend ModelSlim). Validated on Ascend 910B3 at TP=8 + expert parallel with vLLM-Ascend, on
the Partner-Guide Chatbot set (512/256, rate 1–9, 0 server errors).

## Model identity

| Field | Value |
|---|---|
| Publisher | ZhipuAI (`zai-org`) |
| Architecture | GLM-5.1 (MoE) |
| Parameters | ~755B total / ~42B active |
| Native dtype | BF16 |
| Quantization | **W4A8** (Ascend ModelSlim) — ~391 GB (vs ~1.5 TB BF16) |
| Hugging Face | https://huggingface.co/zai-org/GLM-5.1 |

| Platform | Engine | Version | Status |
|---|---|---|---|
| Ascend 910B3 × 8 (TP=8 + EP, whole cards) | vLLM-Ascend | `v0.19.1rc1-openeuler` + `--enforce-eager` | ✅ Partner-Guide Chatbot, 9-rate sweep, 0 server errors |

## Deploy

| Config | File |
|---|---|
| vLLM-Ascend, TP=8 + EP, W4A8 | [`glm-5-1-w4a8-vllm-ascend-tp8.yaml`](https://github.com/alauda/aml-docs/tree/master/docs/en/inference_guide/assets/glm-5-1-w4a8/glm-5-1-w4a8-vllm-ascend-tp8.yaml) |

```bash
base=https://raw.githubusercontent.com/alauda/aml-docs/master/docs/en/inference_guide/assets/glm-5-1-w4a8
# edit namespace / image tag / storageUri first, then:
kubectl apply -f $base/glm-5-1-w4a8-vllm-ascend-tp8.yaml
```

:::warning Two deployment requirements
- **Image must be `>= v0.19.1rc1-openeuler`.** `v0.18.0` ships transformers 4.57.4, too old for
  the GLM-5.1 `glm_moe_dsa` model type (needs transformers ≥ 5.2.0).
- **`--enforce-eager` is required.** A ≥512-token prefill otherwise triggers an AI Core fault
  in the W4A8 dynamic-quant kernel (`aclnnDynamicQuantV2`); eager mode avoids it.
:::
